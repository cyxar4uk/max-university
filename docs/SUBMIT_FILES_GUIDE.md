### Раздел 1: Docker-образ

**Описание:**

Проект использует Docker для контейнеризации приложения. Предоставлены Dockerfile для backend (FastAPI) и frontend (React + Nginx).

**Команды для сборки и запуска:**

```bash
# Сборка и запуск всех сервисов
docker-compose up --build

# Запуск в фоновом режиме
docker-compose up -d --build

# Остановка
docker-compose down
```

**Сборка образов по отдельности:**

Backend:
```bash
docker build -t max-university-backend -f Dockerfile .
docker run -p 8000:8000 max-university-backend
```

Frontend:
```bash
docker build -t max-university-frontend -f Dockerfile.frontend .
docker run -p 80:80 max-university-frontend
```

**После запуска:**
- Frontend: http://localhost
- Backend: http://localhost:8000
- API документация: http://localhost:8000/docs

### Раздел 2: requirements.txt (Зависимости Python)

```
fastapi==0.104.1
uvicorn==0.24.0
pydantic==2.5.0
httpx==0.25.2
python-multipart==0.0.6
```

**Описание библиотек:**

- **fastapi**: Современный web-framework для создания API с автоматической документацией
- **uvicorn**: ASGI сервер для запуска FastAPI с поддержкой async/await
- **pydantic**: Валидация данных и сериализация с использованием типов Python
- **httpx**: Async HTTP клиент для взаимодействия с MAX API
- **python-multipart**: Обработка multipart/form-data запросов

### Раздел 3: Инструкция по запуску

**Вариант 1: Локальный запуск без Docker**

**Требования:**
- Node.js 18+
- Python 3.9+
- npm

**Шаги:**

1. Клонировать репозиторий:
```bash
git clone https://github.com/cyxar4uk/max-university.git
cd max-university
```

2. Установить зависимости фронтенда:
```bash
npm install
```

3. Установить зависимости бэкенда:
```bash
python -m venv venv
venv\Scripts\activate  # Windows
source venv/bin/activate  # Linux/Mac
pip install -r requirements.txt
```

4. Запустить бэкенд (терминал 1):
```bash
venv\Scripts\activate
python main.py
```

Backend доступен на: http://localhost:8000

5. Запустить фронтенд (терминал 2):
```bash
npm run dev
```

Frontend доступен на: http://localhost:3000

---

**Вариант 2: Запуск через Docker Compose**

**Требования:**
- Docker Desktop (Windows/Mac) или Docker Engine (Linux)
- Docker Compose

**Шаги:**

1. Убедиться что Docker Desktop запущен

2. Клонировать репозиторий:
```bash
git clone https://github.com/cyxar4uk/max-university.git
cd max-university
```

3. Запустить через Docker Compose:
```bash
docker-compose up --build
```

4. Открыть приложение:
- Frontend: http://localhost
- Backend API: http://localhost:8000
- API документация: http://localhost:8000/docs

5. Остановка:
```bash
docker-compose down
```

### Раздел 4: Структура проекта

**Backend:**
- main.py - FastAPI сервер с MAX Bot API (1323 строки)
- database.py - Работа с SQLite базами данных (640+ строк)
- data/ - Папка с файлами .db (создается автоматически)

**Frontend:**
- App.v2.jsx - Главный компонент с роутингом
- 15 страниц для разных разделов
- 10 виджетов для блоков
- api-service.js - API клиент с мок-режимом
- styles.css - Кастомные стили (1100+ строк)

**Конфигурация:**
- package.json - Node.js зависимости
- vite.config.js - Конфигурация сборщика
- docker-compose.yml - Оркестрация сервисов

**База данных:**
- users.db - Пользователи и суперадмины
- universities.db - Университеты и коды приглашения
- config.db - Разделы, блоки, кастомные виджеты
