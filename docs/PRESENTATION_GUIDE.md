### Слайд 5: Технические требования к итоговому решению

**Backend**:
- Python 3.9+ (async/await для высокой производительности)
- FastAPI 0.104+ (современный API framework)
- SQLite (для MVP, легко мигрируется на PostgreSQL)
- Uvicorn (ASGI сервер)

**Frontend**:
- React 18 (декларативный UI)
- Vite 5 (быстрая разработка)
- Redux Toolkit (управление состоянием)
- Собственные стили (без внешних зависимостей)

**Интеграция**:
- MAX Bridge API (взаимодействие с мессенджером)
- MAX Bot API (чат-бот)
- MAX UI guidelines (единый стиль)

**Инфраструктура**:
- Docker + Docker Compose (контейнеризация)
- GitHub Actions (CI/CD)
- GitHub Pages (хостинг frontend)

**Безопасность**:
- HTTPS (обязательно для MAX Bridge)
- CORS настройки
- Валидация данных (Pydantic)
- Санитизация пользовательского кода

**Масштабируемость**:
- Горизонтальное масштабирование (за load balancer)
- Кеширование (Redis)
- Очереди задач (Celery)

### Слайд 6: Ключевые фичи и пользовательские сценарии

**1. Система кодов приглашения**

Сценарий:
```
Администратор → Генерирует код для студента
              ↓
Студент → Получает код (email/sms/бумага)
        ↓
        Открывает мини-приложение
        ↓
        Вводит код
        ↓
        Автоматически попадает в систему вуза со своей ролью
```

**Преимущества**:
- Безопасность (только с кодом)
- Контроль администрации
- Невозможность подделки роли
- Массовая генерация и импорт

**2. Конструктор интерфейсов**

Сценарий:
```
Администратор вуза:
  1. Открывает админ-панель
  2. Выбирает роль (студент/абитуриент/...)
  3. Создает разделы ("Учеба", "Жизнь", ...)
  4. Добавляет блоки в разделы (расписание, новости, ...)
  5. Перетаскивает для изменения порядка
  6. Настраивает цвет хедера
  7. Сохраняет → студенты видят изменения моментально
```

**Преимущества**:
- Без программирования
- Уникальность для каждого вуза
- Гибкость настроек
- Быстрые изменения

**3. Виджеты вместо ссылок**

Обычное приложение:
```
[Кнопка "Расписание"] → клик → загрузка → просмотр
```

Наше решение:
```
[Виджет расписания]
  ├─ 9:00 - Математика (ауд. 301)
  ├─ 10:45 - Физика (ауд. 205)
  └─ 12:30 - История (ауд. 410)
```

**Реализованные виджеты**:
- Расписание (календарь на сегодня)
- Новости (карусель с автопрокруткой)
- Услуги (сетка быстрых действий)
- Курсы (с прогресс-барами)
- События (список с регистрацией)
- Оплата (баланс + кнопка оплаты)
- Аналитика (карточки статистики)
- Цифровой пропуск (QR-код с таймером)

**4. Кастомные блоки с модерацией**

Сценарий:
```
Администратор вуза:
  1. Хочет уникальную функцию (например, "Рейтинг преподавателей")
  2. Разрабатывает виджет по стандартам (или заказывает)
  3. Отправляет код на модерацию через интерфейс
  4. Суперадмин проверяет безопасность
  5. Одобряет → виджет доступен вузу
```

**Преимущества**:
- Безграничная кастомизация
- Безопасность (проверка кода)
- Экосистема виджетов
- Монетизация (в будущем)

**5. Мок-режим для демонстрации**

Сценарий:
```
Пользователь открывает на GitHub Pages
  ↓
Backend недоступен
  ↓
Автоматическое переключение на мок-данные
  ↓
Показ уведомления "Работа в демо-режиме"
  ↓
Полный функционал доступен с тестовыми данными
```

**Преимущества**:
- Всегда работает
- Легкая демонстрация
- Тестирование без backend
- Отладка через логи
