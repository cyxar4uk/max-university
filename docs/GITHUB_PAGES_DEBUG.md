# Отладка GitHub Pages

## Проблема: Белый экран на GitHub Pages

Если приложение не загружается на https://cyxar4uk.github.io/max-university/, выполните следующие шаги:

## Шаг 1: Проверка консоли браузера

1. Откройте https://cyxar4uk.github.io/max-university/
2. Нажмите F12 (открыть DevTools)
3. Перейдите на вкладку **Console**
4. Проверьте ошибки

### Типичные ошибки:

#### Ошибка 404 для main.jsx или других файлов
**Причина:** Неправильные пути к файлам
**Решение:** Убедитесь, что `base: '/max-university/'` установлен в `vite.config.js`

#### Ошибка "Failed to fetch" или CORS
**Причина:** Проблемы с загрузкой модулей
**Решение:** Проверьте, что все импорты имеют правильные расширения

#### Ошибка "Cannot find module"
**Причина:** Отсутствующие файлы или неправильные импорты
**Решение:** Проверьте все импорты в файлах

## Шаг 2: Проверка Network вкладки

1. В DevTools перейдите на вкладку **Network**
2. Обновите страницу (Ctrl+F5)
3. Проверьте, какие файлы не загружаются (красным цветом)

### Что должно загружаться:
- `index.html` - статус 200
- `main.jsx` или `main-[hash].js` - статус 200
- `assets/index-[hash].js` - статус 200
- `assets/index-[hash].css` - статус 200

## Шаг 3: Проверка сборки локально

```bash
# Соберите проект локально
npm run build

# Проверьте содержимое папки dist/
# Должны быть файлы:
# - index.html
# - assets/ (с JS и CSS файлами)
```

## Шаг 4: Проверка GitHub Actions

1. Перейдите на https://github.com/cyxar4uk/max-university/actions
2. Проверьте последний workflow "Deploy to GitHub Pages"
3. Убедитесь, что сборка прошла успешно (зеленая галочка)

### Если сборка не удалась:
- Проверьте логи ошибок
- Убедитесь, что все зависимости установлены
- Проверьте синтаксис всех файлов

## Шаг 5: Проверка настроек GitHub Pages

1. Перейдите в Settings → Pages
2. Убедитесь, что:
   - Source: **GitHub Actions**
   - Branch: **main** или **master**

## Шаг 6: Очистка кеша

Иногда проблема в кеше браузера:

1. **Очистите кеш браузера:**
   - Chrome/Edge: Ctrl+Shift+Delete
   - Firefox: Ctrl+Shift+Delete
   - Выберите "Кешированные изображения и файлы"

2. **Или откройте в режиме инкогнито:**
   - Chrome/Edge: Ctrl+Shift+N
   - Firefox: Ctrl+Shift+P

3. **Или добавьте параметр к URL:**
   ```
   https://cyxar4uk.github.io/max-university/?v=2
   ```

## Шаг 7: Проверка файлов в репозитории

Убедитесь, что все необходимые файлы закоммичены:

```bash
git status
git add .
git commit -m "Fix imports and paths"
git push
```

## Шаг 8: Принудительный пересбор

Если ничего не помогает:

1. Сделайте пустой коммит:
   ```bash
   git commit --allow-empty -m "Force rebuild"
   git push
   ```

2. Это запустит GitHub Actions заново

## Типичные проблемы и решения

### Проблема: "Cannot GET /max-university/"
**Решение:** Убедитесь, что `index.html` находится в корне `dist/` после сборки

### Проблема: "404 для всех JS файлов"
**Решение:** Проверьте `base: '/max-university/'` в `vite.config.js`

### Проблема: "Модуль не найден"
**Решение:** Добавьте расширения `.js` или `.jsx` ко всем импортам

### Проблема: "Белый экран без ошибок"
**Решение:** 
1. Проверьте, что React приложение монтируется в `#root`
2. Проверьте консоль на наличие ошибок рендеринга
3. Убедитесь, что `main.jsx` правильно импортирует `App.v2.jsx`

## Проверка работоспособности

После исправлений проверьте:

1. ✅ Открывается страница без белого экрана
2. ✅ Видна форма ввода кода приглашения
3. ✅ Консоль не показывает критических ошибок
4. ✅ Все файлы загружаются (Network вкладка)
5. ✅ Приложение реагирует на действия пользователя

## Если ничего не помогает

1. Проверьте, что локально все работает:
   ```bash
   npm run dev
   # Откройте http://localhost:3000
   ```

2. Если локально работает, но на GitHub Pages нет:
   - Проблема в конфигурации сборки
   - Проверьте `vite.config.js`
   - Проверьте `.github/workflows/deploy.yml`

3. Создайте issue в репозитории с:
   - Скриншотом консоли браузера
   - Логами GitHub Actions
   - Описанием проблемы
