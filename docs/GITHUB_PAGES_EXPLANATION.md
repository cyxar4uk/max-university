# Почему GitHub Pages не работает с бэкендом

## Проблема

**GitHub Pages - это статический хостинг**, который может размещать только:
- HTML файлы
- CSS стили
- JavaScript файлы (клиентский код)
- Изображения и другие статические ресурсы

**GitHub Pages НЕ МОЖЕТ:**
- Запускать серверные приложения (Python, Node.js и т.д.)
- Выполнять код на сервере
- Хранить базы данных
- Обрабатывать POST/PUT/DELETE запросы к серверу

## Текущая ситуация

Ваше приложение **УЖЕ РАБОТАЕТ** на GitHub Pages, но использует **мок-данные** вместо реального бэкенда.

### Как это работает сейчас:

1. Фронтенд загружается с GitHub Pages ✅
2. Приложение пытается подключиться к бэкенду по адресу `http://localhost:8000/api`
3. Бэкенд недоступен (его нет на GitHub Pages)
4. Приложение автоматически переключается на **мок-данные** ✅
5. Показывается уведомление о работе в демо-режиме
6. Все функции работают с тестовыми данными

## Решения

### Вариант 1: Оставить как есть (для демонстрации)

**Плюсы:**
- Работает прямо сейчас на GitHub Pages
- Не требует дополнительных настроек
- Все функции доступны с мок-данными

**Минусы:**
- Нет реальной базы данных
- Данные не сохраняются между сессиями
- Нет реальной интеграции с MAX API

**Подходит для:** Демонстрации проекта на хакатоне

### Вариант 2: Развернуть бэкенд отдельно (для продакшена)

Нужно разместить бэкенд на отдельном хостинге:

#### Бесплатные варианты:

1. **Railway** (https://railway.app)
   - Бесплатный тариф с ограничениями
   - Простое развертывание из GitHub
   - Автоматический HTTPS

2. **Render** (https://render.com)
   - Бесплатный тариф (засыпает после 15 минут бездействия)
   - Простое развертывание
   - Автоматический HTTPS

3. **Fly.io** (https://fly.io)
   - Бесплатный тариф
   - Хорошая производительность
   - Docker поддержка

4. **Heroku** (https://heroku.com)
   - Платный (но есть бесплатные альтернативы)
   - Простое развертывание

#### Что нужно сделать:

1. **Развернуть бэкенд на хостинге:**
   ```bash
   # Например, на Railway:
   # 1. Подключить GitHub репозиторий
   # 2. Выбрать main.py как точку входа
   # 3. Установить переменные окружения:
   #    - SECRET_KEY
   #    - MAX_BOT_TOKEN
   ```

2. **Обновить URL бэкенда в GitHub Actions:**
   
   Отредактировать `.github/workflows/deploy.yml`:
   ```yaml
   env:
     VITE_API_URL: https://your-backend-url.railway.app/api
   ```

3. **Настроить CORS на бэкенде:**
   
   В `main.py` уже есть CORS настройки, но нужно добавить ваш GitHub Pages домен:
   ```python
   origins = [
       "https://cyxar4uk.github.io",
       "http://localhost:3000",
       # ... другие домены
   ]
   ```

4. **Пересобрать и задеплоить фронтенд:**
   ```bash
   git commit -am "Update backend URL"
   git push
   ```

### Вариант 3: Использовать Serverless функции

Можно переписать бэкенд на serverless функции:
- **Vercel Functions** (для фронтенда на Vercel)
- **Netlify Functions**
- **Cloudflare Workers**

Но это требует рефакторинга кода.
